# Contributing to MetaSkillBase-Core



Thank you for your interest in contributing to MetaSkillBase-Core!



## Who Can Contribute?



- **Contributor**: Anyone who submits skills, demands, code, or documentation

- **Developer**: Anyone who improves the Cutter Engine or fixes bugs

- **Documenter**: Anyone who updates docs, fixes typos, or adds examples



## Contribution Workflow



### 1. Submit New Demand



1. Create a new Issue in the repository

2. Title format: Demand: [Skill Name] – [Description]

3. Clearly describe the demand

4. Add label: demand



### 2. Develop Skill



1. Select an unclaimed Issue with demand label

2. Comment on the Issue to claim

3. Develop the skill following atomic rules

4. Place the skill in the correct category directory

5. Create a new PR with the skill files



### 3. Contribute to Cutter Engine



1. Follow standard code development practices

2. Follow Python PEP 8 code style

3. Add comments for core logic and new features

4. Test changes locally



## Contribution Standards



### 1. Skill Standards



- **Atomic**: One skill = one minimal reusable function

- **Naming**: Use lowercase letters and hyphens

- **Compatibility**: All skills are OpenClaw compatible

- **Documentation**: Every skill must have a Markdown file



### 2. Code Standards



- Follow Python PEP 8

- Use meaningful variable/function/class names

- Add docstrings



## Recognition



All contributions are recorded in the skill index, including the contributor's GitHub ID. Your skill's Usage Count reflects your contribution value to the community!

# 为 MetaSkillBase-Core 贡献



感谢您有兴趣为 MetaSkillBase-Core 贡献！



## 谁能贡献？



- **贡献者**：提交技能、需求、代码或文档的任何人

- **开发者**：改进Cutter引擎或修复bug的任何人

- **文档撰写者**：更新文档、纠正错别字或添加示例的任何人



## 贡献工作流



### 1. 提交新需求



1. 在仓库中创建新Issue

2. 标题格式：Demand: [技能名称] – [描述]

3. 清晰描述需求

4. 添加标签：demand



### 2. 开发技能



1. 选择带有demand标签的未认领Issue

2. 在Issue上评论认领

3. 遵循原子化规则开发技能

4. 将技能放置在正确的分类目录

5. 创建包含技能文件的新PR



### 3. 为Cutter引擎贡献



1. 遵循标准代码开发实践

2. 遵循Python PEP 8代码风格

3. 为核心逻辑和新功能添加注释

4. 本地测试更改



## 贡献标准



### 1. 技能标准



- **原子性**：一技能 = 最小可复用功能

- **命名**：使用小写字母和连字符

- **兼容性**：所有技能与OpenClaw兼容

- **文档**：每个技能必须有Markdown文件



### 2. 代码标准



- 遵循Python PEP 8

- 使用有意义的变量/函数/类名

- 添加文档字符串



## 认可



所有贡献都记录在技能索引中，包含贡献者的GitHub ID。您的技能使用次数反映您对社区的贡献价值！
