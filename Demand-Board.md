# Demand Board



Rules and workflow for submitting, claiming, and implementing demands.



## 1. Demand Submission



- Submit clear, executable human demands

- Title format: Demand: [Skill Name] – [Description]

- Label: demand

- You will be marked as the contributor for that demand



## 2. Skill Implementation



- Claim unclaimed demands

- Develop atomic skills following core rules

- Place in correct category

- You will be marked as the contributor for that skill



## 3. Cutter Integration



When a demand is submitted:

1. Cutter searches OpenClaw community for Bot Skills

2. Cutter queries local Skills

3. Cutter searches MetaSkillBase-Core for Meta Skills

4. Missing Meta Skills are recorded for development



## 4. Workflow



Submit Demand → Claim → Develop → Merge → Track Usage Count

# Demand-Board.md 中文翻译



# 需求看板



提交、认领和实施需求的规则和工作流程。



## 1. 需求提交



- 提交清晰、可执行的人类需求

- 标题格式：Demand: [技能名称] – [描述]

- 标签：demand

- 您将被标记为该需求的贡献者



## 2. 技能实施



- 认领未认领的需求

- 遵循核心规则开发原子化技能

- 放置在正确的分类中

- 您将被标记为该技能的贡献者



## 3. Cutter集成



当提交需求时：

1. Cutter搜索OpenClaw社区的Bot Skills

2. Cutter查询本地Skills

3. Cutter搜索MetaSkillBase-Core的Meta Skills

4. 缺失的Meta Skills被记录用于开发



## 4. 工作流程



提交需求 → 认领 → 开发 → 合并 → 跟踪使用次数
