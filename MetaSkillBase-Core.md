# MetaSkillBase-Core

## Core Specification & Repository Rules



### 1. Core Philosophy

- One for Human, One for Bot.

- Elastic Classification: No rejection, no audit, only organization.

- Usage Count is a community metric reflecting skill value, recorded and maintained by contributors.

- OpenClaw Compatible: All skills work with OpenClaw.



### 2. Skill Hierarchy

- **Meta Skill**: Finest-grained reusable functional component.
- **Bot Skill**: High-level solution assembled from Meta Skills.



### 3. Repository Structure

All skills are organized into 6 fixed categories:

- 00-Unclassified: Default for unclassified skills

- 01-Base: Fundamental atomic operations

- 02-Device: Device & system operations

- 03-Manual: Machine-readable manuals

- 04-Process: Enterprise process units

- 05-Interaction: Human-bot collaboration actions



### 4. Contribution Rules

- Any demand or skill can be submitted.

- Skills must be atomic: one skill = one minimal function.

- All skills are OpenClaw compatible.

- All contributions are recorded and credited.



### 5. Cutter Engine Rules

- Search OpenClaw community for Bot Skills

- Query local Skills

- Search MetaSkillBase-Core for Meta Skills

- Submit development demand for missing Meta Skills

# MetaSkillBase-Core

## 核心规范与仓库规则



### 1. 核心哲学

- 一人一机。

- 弹性分类：无拒绝、无审核、仅组织。

- 使用次数作为核心价值衡量指标。

- OpenClaw兼容：所有技能均可与OpenClaw配合使用。



### 2. 技能层级

- **Bot Skill**：由多个Meta Skills组成的复杂技能

- **Meta Skill**：形成Bot Skill的原子化可复用技能单元



### 3. 仓库结构

所有技能分为6个固定类别：

- 00-Unclassified：未分类技能的默认位置

- 01-Base：基础原子操作

- 02-Device：设备与系统操作

- 03-Manual：机器可读手册

- 04-Process：企业流程单元

- 05-Interaction：人机协作动作



### 4. 贡献规则

- 任何需求或技能都可以提交

- 技能必须原子化：一技能 = 最小功能

- 所有技能与OpenClaw兼容

- 所有贡献都被记录和认可



### 5. Cutter引擎规则

- 搜索OpenClaw社区的Bot Skills

- 查询本地Skills

- 搜索MetaSkillBase-Core的Meta Skills

- 提交缺失Meta Skills的开发需求
